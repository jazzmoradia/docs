OmniScape is built for **Unreal Engine 5.5 and newer**. It works with any art style and imposes no constraints on your mesh pipeline beyond standard Unreal asset requirements. If your meshes work elsewhere in your project, they'll work with OmniScape.

## Engine version

| Requirement | Detail |
|---|---|
| **Minimum engine version** | Unreal Engine 5.5 |
| **Recommended** | Latest stable 5.5.x or newer |
| **Earlier versions (5.3, 5.4)** | Not supported |

If you're upgrading an existing project to 5.5 specifically to use OmniScape, the [Unreal Engine migration guide](https://dev.epicgames.com/documentation/en-us/unreal-engine/upgrading-projects-in-unreal-engine) covers the standard steps. OmniScape has no additional migration requirements beyond the engine upgrade itself.

The **Control Center panel is editor-only** and is stripped from cooked builds automatically. The generated content — instanced meshes, spline actors, Blueprint actors — cooks exactly like any other level content. Players in shipped games do not need the plugin installed.

## Asset compatibility

OmniScape accepts the three standard Unreal asset types for generation slots:

<Tabs>
  <Tab title="Static Meshes">
    Any Static Mesh asset works in the Large, Medium, and Small building tiers, as well as road, wall, and pillar slots. No special import settings are required beyond the collision notes below.
  </Tab>
  <Tab title="Blueprint Classes">
    Any Blueprint class that extends `Actor` can be assigned to building tiers. OmniScape spawns and places these actors at generation time. Gameplay logic inside the Blueprint is not invoked during generation — it runs as normal once the level is played.
  </Tab>
  <Tab title="Foliage Types">
    Foliage Type assets are used in the Ambient tab's flora layer. Any Foliage Type configured for your meshes works. OmniScape distributes these using standard Instanced Static Mesh spawning, so HISM performance budgets apply normally.
  </Tab>
</Tabs>

## Collision requirements

OmniScape uses collision for terrain-following road elevation and for surface placement of ambient layers. Two requirements apply:

- **CPU collision must be enabled** on any Static Mesh placed on walkable terrain surfaces. This is Unreal's standard "Complex Collision" or a simplified convex hull — either works. GPU-only or no-collision meshes will cause placement and elevation queries to miss.
- **World Static collision channel** must be active on terrain and ground meshes. OmniScape queries this channel during road contouring and ambient scatter. If your terrain uses a custom channel, roads may not follow terrain correctly.

<Warning>
  If roads appear to hover above or sink into your terrain, the most common cause is missing CPU collision on the landscape or ground mesh. Check **Asset Details → Collision Complexity** on the asset in question.
</Warning>

## Optional module dependencies

OmniScape's core generation does not require any optional Unreal modules. The plugin compiles and runs without them. However, three modules unlock additional features when present:

| Module | What it enables | Required? |
|---|---|---|
| **PCG (Procedural Content Generation)** | PCG Hand-off component for exporting layout data to external PCG graphs | Optional |
| **MetaSounds** | Full MetaSounds graph support in the Ambient audio layer | Optional (standard Sound Cues work without it) |
| **Landscape** | Landscape paint bake in the Structure tab | Optional (no effect if your scene has no Landscape actor) |

If a module is absent, the relevant UI section in the Control Center is hidden or disabled. No errors are thrown and no other features are affected.

## Editor vs. cooked build summary

| Component | Editor | Cooked build |
|---|---|---|
| Control Center panel | ✅ Present | ❌ Stripped (editor-only) |
| OmniScape Generator actor | ✅ Active | ✅ Present (inert after Save to Level) |
| Generated instances and actors | ✅ Live | ✅ Cook as normal level content |
| Blueprint API | ✅ Available | ✅ Available for runtime generation |
| Preset files | ✅ Available | ❌ Not needed in cooked build |

<Tip>
  After generating, use **Save to Level** (or simply save the level) to convert generated content into standard level actors. From that point on, the level is self-contained and packages without the Control Center.
</Tip>
