## Engine versions

OmniScape runs on a single codebase validated across modern Unreal Engine releases.

<Info>
  **Unreal Engine 5.5 – 5.8.** The Control Center is an editor tab; generated content cooks like any other placed actor or instanced component.
</Info>

| | |
|---|---|
| **Minimum engine** | Unreal Engine 5.5 |
| **Validated** | UE 5.5, 5.7, and 5.8 (v3.0) |
| **Control Center** | Editor-only (Slate panel) |
| **Generated content** | Cooks normally once saved to the level |

## Optional engine plugins

Core generation never depends on optional modules. OmniScape compiles and runs without them, lighting up extra capability when they're present.

| Plugin | Unlocks | Required? |
|---|---|---|
| **PCG** | PCG Hand-off export <span className="os-tag os-phase-ambient">Beta</span> | No |
| **MetaSounds** | Ambisonic ambient audio modes <span className="os-tag os-phase-ambient">Beta</span> | No — falls back to positional audio |
| **Landscape** | Landscape paint bake & road deform <span className="os-tag os-phase-ambient">Beta</span> | No |

## Platforms

Targets match the published plugin list (desktop and mobile where enabled). Because the Control Center is editor-only, your **saved** settlement content packages like any other level content.

## Packaging at a glance

<Steps>
  <Step title="Generate in the editor">
    Tune with the live preview, then click **Generate** when the composition is right.
  </Step>
  <Step title="Save to Level">
    Persist the generated actors and instanced components into the map.
  </Step>
  <Step title="Cook">
    Package as normal. Players see the saved content — no Control Center, no runtime generation required.
  </Step>
</Steps>

<Tip>
  Runtime generation from Blueprint is supported for special cases, but **bake-first** is recommended for shipped games. See [Blueprint & runtime](/guides/blueprint-and-runtime).
</Tip>

<Note>
  **Fab currently ships 2.7.0.** Engine validation up to 5.8 and the v3 feature set ship with the **3.0.0** release.
</Note>
