## Installation & upgrades

<AccordionGroup>
  <Accordion title="My built-in presets are missing">
    Install the latest version from [Fab](https://fab.com/s/4aeff49faf04). Built-in presets ship with the plugin; a partial install leaves the list empty.
  </Accordion>
  <Accordion title="My layout changed after updating">
    **Legacy mode** (an advanced road option) restores older road behaviour for the same seed. New work stays on the current road stack. Built-in presets target the current baseline for everyone.
  </Accordion>
  <Accordion title="I build from source / edit C++">
    After changing struct or class layout, close the editor and do a full rebuild — don't rely on Live Coding for plugin internals.
  </Accordion>
</AccordionGroup>

## Editor workflow

<AccordionGroup>
  <Accordion title="Generate vs Auto-Generate">
    Use **Generate** for heavy maps. **Auto-Generate** rebuilds full geometry on every edit. The live preview updates either way, so tune with the preview and Generate when ready.
  </Accordion>
  <Accordion title="My settlement disappeared">
    Save after generating. See [the one rule](/get-started/save-to-level).
  </Accordion>
  <Accordion title="I moved the Generator and it looks wrong">
    The settlement is centred on the actor. Move it, regenerate, and Save to Level.
  </Accordion>
</AccordionGroup>

## Assets & placement

<AccordionGroup>
  <Accordion title="Nothing spawns after Generate">
    Add a **Large** asset — it's required, one landmark per node.
  </Accordion>
  <Accordion title="The same mesh in two tiers — is that OK?">
    Yes. Weights are still applied per tier.
  </Accordion>
  <Accordion title="Blueprint and mesh on one row">
    The Blueprint wins for that pick; the mesh is the fallback when there's no Blueprint on the chosen entry.
  </Accordion>
  <Accordion title="Flora asset types — Static Mesh, Foliage Type, or Blueprint?">
    **Static Mesh** uses the generator's HISM (fast, default). **Foliage Type** spawns into Unreal's foliage system for harvest/interaction and Foliage Mode. **Blueprint** spawns actors. Priority when multiple are set: Blueprint → Foliage Type → Mesh.
  </Accordion>
  <Accordion title="My building has a bad footprint">
    Use **Bounds override** to fix a wrong pivot or bounds.
  </Accordion>
</AccordionGroup>

## Roads & walls

<AccordionGroup>
  <Accordion title="Optional v3.0 asset slots">
    With v3.0 you can assign extra meshes — per-class road meshes (Arterial/Collector/Local), tower/bastion meshes, and a foundation/plinth. Empty slots are silently skipped; none are required for the standard look. <span className="os-tag os-phase-structure">v3.0</span>
  </Accordion>
  <Accordion title="My intersection mesh does nothing">
    It needs the current pipeline, an assigned mesh, and a sensible radius scale. It isn't stored inside presets — set it after loading.
  </Accordion>
  <Accordion title="Only the walls changed">
    Try **Rebuild walls only**; run a full Generate if anything else looks stale.
  </Accordion>
</AccordionGroup>

## Packaging & runtime

<AccordionGroup>
  <Accordion title="How do I ship a settlement?">
    Bake-first: generate in the editor, **Save to Level**, then cook. Players see saved meshes and instances with no hitch.
  </Accordion>
  <Accordion title="Can I generate at runtime?">
    Yes — call **GenerateSettlement** from Blueprint in cooked builds. Expect a frame hitch unless you enable **Time-sliced spawn**. Use bake **or** runtime generate, not both on the same layout. See [Blueprint & runtime](/guides/blueprint-and-runtime).
  </Accordion>
  <Accordion title="What's inside a preset?">
    Numbers and toggles — settlement config, pattern, roads, walls, counts, flora samples, the zone table, and more. **Not** inside: mesh lists, volumes, anchors, road asset lists, intersection mesh, or wall mesh lists.
  </Accordion>
</AccordionGroup>
