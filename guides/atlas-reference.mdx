Atlas is OmniScape's real-world map import wizard. It lets you feed an actual place — a medieval city center, a coastal fishing village, a desert trade hub — into the layout engine as a structural reference, giving your settlement the organic irregularity of somewhere that genuinely grew over time. Atlas is not photogrammetry or satellite imagery: it reads street network geometry and translates it into OmniScape road nodes, not textures or elevation data. Your asset library, material palette, and art direction remain fully in control of the final look; Atlas only informs the shape of the plan.

<Note>
  The full Atlas workflow, including advanced import options and troubleshooting, is documented in **Docs/OmniScape_Atlas.md**, which ships with the plugin inside your project's Plugins folder.
</Note>

## Opening Atlas

Atlas lives inside the Control Center, which is an editor-only tool.

<Steps>
  <Step title="Open the Control Center">
    In the Unreal Editor toolbar, click the **OmniScape** button to open the Control Center, then select the generator actor you want to configure.
  </Step>
  <Step title="Navigate to the Atlas wizard">
    Go to **Layout → Atlas**. The wizard opens with three sequential stages displayed at the top: **Locate → Tune → Generate**.
  </Step>
  <Step title="Work through the stages">
    Complete each stage in order. Locate finds or loads your reference data, Tune adjusts how that data is interpreted, and Generate applies it and builds the settlement.
  </Step>
</Steps>

## Importing a location

The most direct path into Atlas is a place name or a map link.

<Steps>
  <Step title="Enter a location">
    In the **Locate** stage, type a place name (for example, `Carcassonne, France` or `Dubrovnik old town`) or paste a direct map link from a supported provider into the search field.
  </Step>
  <Step title="Import">
    Click **Import This Location**. OmniScape fetches the street network data for that area and displays a preview. The **© OpenStreetMap contributors** attribution notice appears automatically — see [Attribution requirement](#attribution-requirement) below.
  </Step>
  <Step title="Choose fidelity and layout source">
    In the **Tune** stage, select the fidelity level that matches your settlement's scale and detail needs, and choose which layer of the imported data to use as the primary layout source (street centerlines, block boundaries, etc.).
  </Step>
  <Step title="Generate">
    Click **Generate Settlement**. OmniScape uses the imported reference as the structural basis for road placement, then runs the full pipeline — buildings, flora, walls — on top of that layout using your configured assets and parameters.
  </Step>
</Steps>

<Tip>
  Zoom the map preview in the Locate stage to a neighborhood or district rather than an entire city. A tighter import area gives cleaner, more usable node density for typical settlement sizes (radius 5 000 – 20 000 cm). Full cities at low fidelity often collapse to a handful of nodes.
</Tip>

## Offline import

If you are working without internet access, or want to use data you have curated yourself, Atlas supports three offline input types:

<CardGroup cols={3}>
  <Card title=".osm files" icon="file-code">
    Export street data from OpenStreetMap's web export tool or Overpass API and save the `.osm` file locally. Use **Import from file** in the Locate stage and select your file.
  </Card>
  <Card title=".geojson files" icon="map">
    Any GeoJSON file containing `LineString` or `MultiLineString` features representing roads or paths can be imported. Useful for custom-drawn reference networks authored in GIS tools.
  </Card>
  <Card title="Screenshot reference" icon="image">
    Drop a top-down map image (a historical map, a hand-drawn plan, a game map scan) as a visual reference overlay. OmniScape does not trace the image automatically — you use it as a visual guide while adjusting layout parameters manually in the Tune stage.
  </Card>
</CardGroup>

## Attribution requirement

Data imported via the online place-name search and map link routes is sourced from **OpenStreetMap**. OpenStreetMap's license (ODbL) requires attribution wherever the derived data is publicly displayed.

<Warning>
  Any product — game, application, or interactive experience — that uses layouts generated from OpenStreetMap data must include the text **© OpenStreetMap contributors** in a visible location (credits screen, in-game attribution panel, or similar). This is a legal requirement of the ODbL license, not optional.
</Warning>

OmniScape displays the attribution string automatically in the Atlas wizard after a successful import as a reminder. It is your responsibility to carry that attribution through to your shipped product. Layouts generated from offline `.osm` exports carry the same requirement if the source data originated from OpenStreetMap.

Layouts generated from your own `.geojson` data or from screenshot reference only (no OSM data involved) are not subject to this requirement.

## Diagnostics and advanced import settings

For import problems — missing roads, unexpected scale, disconnected nodes — open the **Details → Atlas** panel within the Control Center. It exposes:

- Raw import statistics (node count, edge count, bounding box)
- Per-layer visibility toggles for inspecting what was imported
- Advanced fidelity and simplification controls for fine-tuning how the street network is converted to OmniScape nodes
- Re-import options without leaving the Tune stage

If an import produces a layout that looks nothing like the source location, check the bounding box scale in the diagnostics panel first — very large city-scale imports are often the cause.

---

<Tip>
  Atlas is a **layout reference and inspiration tool**, not a recreation pipeline. Two settlements generated from the same real-world import with different asset libraries, pattern parameters, or fidelity settings will look and feel completely different. The street geometry gives you authentic organic structure; your art direction, biome, and asset choices define the world your players actually see.
</Tip>
