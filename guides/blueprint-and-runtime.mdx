The Control Center is editor-only, but generation is fully scriptable. For shipped games, **bake-first** is recommended; runtime generation is there for tools and special cases.

## Events

| Event | Fires when |
|---|---|
| **OnSettlementGenerated** | A full generate completes (legacy). |
| **OnSettlementGenerationCompleted** | A full generate completes, with a structured result. |
| **OnSettlementCleared** | After the settlement is cleared. |
| **OnGenerationProgress** | Optional 0–1 progress during time-sliced spawn. |
| **On Building Spawned** | For Large/Medium/Small/Sprawl **Blueprint** buildings only (not instanced meshes or environment). |

## Actions

| Action | Role |
|---|---|
| **GenerateSettlement** | Clear, then run the full pipeline. |
| **CanGenerateSettlement** | Preflight check only. |
| **GetLastGenerationResult** | Structured outcome of the last generate. |
| **LoadProfileAndGenerate** | Apply a runtime profile, then generate. |
| **CancelGeneration** | Stop a time-sliced spawn. |
| **ClearSettlement** | Remove generated content. |
| **RebuildWallsOnly** | Rebuild walls from the current layout (needs physics + nodes). |

## Seeding

`GenerationSeed` drives all variation. In the editor, type a seed or **Randomize seed**. At runtime, set `GenerationSeed`, then call `GenerateSettlement`. There's no randomize node at runtime — use your own RNG and write the result into `GenerationSeed`.

## Runtime generation

<Steps>
  <Step title="Preflight">
    Call **CanGenerateSettlement** first. By default, warnings (such as no Large assets) don't block generation — roads, environment, and walls can still run. Set **Require preflight success** to hard-block.
  </Step>
  <Step title="Generate">
    Set `GenerationSeed`, then call **GenerateSettlement**.
  </Step>
  <Step title="Avoid the hitch">
    Enable **Time-sliced spawn** (OmniScape | Runtime) to spread spawning across frames. Time-slicing runs in game worlds only; editor Generate stays a single synchronous call for undo safety.
  </Step>
</Steps>

<Warning>
  Use **bake** or **runtime generate**, not both on the same layout. If a level already contains a baked settlement, generating again at runtime can duplicate content — Clear Settlement removes generator-owned instances, not every saved child actor.
</Warning>

## Profiles

An optional **RuntimeProfile** data asset copies preset-style numbers (not meshes). **LoadProfileAndGenerate** applies it and generates — handy for runtime difficulty or biome variants over the same kit.

## PCG hand-off <span className="os-tag os-phase-ambient">Beta</span>

Enable the PCG data provider on the **Ambient** tab to export nodes, roads, walls, footprints, and bounds as tagged point records for your own PCG graphs. Starter JSON contracts ship in `Content/PCG/`.

<Note>
  Multiplayer: most teams ship baked maps. If you generate at runtime, decide who runs it and budget the hitch; keep `GenerationSeed` identical across clients for matching layout, and treat the output like normal placed actors.
</Note>
