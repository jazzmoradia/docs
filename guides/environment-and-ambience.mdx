A settlement that ends at its last building feels unfinished. The **Ambient** tab fills the space around and between structures — trees pressing in at the edge, rocks breaking up clearings, scattered debris near workshops, distant birdsong, and the low hum of a busy market. These systems layer on top of your layout to turn a generator output into a world.

<Note>
**Fauna**, **Ambient Audio**, and **PCG Hand-off** are Beta features. Enable Beta mode from the status bar (bottom-right of the editor) to unlock them.
</Note>

## Flora Layers

Flora is stable and available in all builds. The **unified Flora layers list** in the Ambient tab holds any number of layers, each tagged with a **Kind chip** that sets its clustering and spawning behaviour.

### Kind chips

| Kind | Behaviour |
|------|-----------|
| **Vegetation** | Trees, shrubs, grasses — standard foliage distribution |
| **Rock** | Rock and boulder placement with scatter clustering |
| **Debris** | Clusters near buildings — rubble, clutter, scatter props (v3; no name-based guessing) |

### Asset types per layer

Each layer slot accepts one of three asset types:

| Asset type | Spawn path | Best for |
|------------|------------|----------|
| **Static Mesh** | Generator HISM (fast instancing) | Default trees, rocks, debris — the fastest option |
| **Foliage Type** | Unreal Engine foliage (AInstancedFoliageActor) | Survival games, harvest/interaction logic |
| **Blueprint Class** | Spawned actor | Interactive props, logic-driven objects |

<Tip>
Use **Static Mesh** by default and switch to Foliage Type only when you need Unreal's foliage tools (e.g. harvesting, painting override, LOD settings via the Foliage Type asset). When multiple types are assigned to the same slot, the generator resolves priority in this order: **Blueprint Class → Foliage Type → Static Mesh**.
</Tip>

### Layer controls

| Control | Range | Role |
|---------|-------|------|
| **Density** | 0.01 — 10 | Overall strength of this layer — how many instances spawn |
| **Min distance from buildings** | — | Inner exclusion band; keeps spawns away from structure walls |
| **Max distance from buildings** | — | Outer band; use to concentrate flora near or away from buildings |
| **Scale** | 0.1 — 10 | Uniform instance size multiplier |
| **Force upright** | on / off | Levels instances on slopes (default off; useful for stumps, boulders) |

<Warning>
**Max distance too small** creates a visible "haircut ring" of flora that ends abruptly in open ground. **Max distance too large** (or unconstrained) spreads props across the entire settlement without clustering. Dial both values together to get the read you want — tight debris near buildings, open clearings in the middle.
</Warning>

### Flora sample counts

The **Flora Samples** value controls how many candidate positions OmniScape evaluates per generation. The valid range is **10 – 5,000**; the default is 400.

| Use | Sample count |
|-----|-------------|
| Fast iteration / preview | 10 — 200 |
| Medium maps, typical production | 400 (default) |
| Dense large maps | 800 — 1,500 |
| Maximum (high-end maps) | up to 5,000 |

<Tip>
Work at **10–200 samples** while you're placing buildings and tuning layout. Raise to your target count only when you're doing a final pass on flora distribution. This keeps generation times short during iteration.
</Tip>

## Fauna Layers (Beta)

The **unified Fauna layers list** works similarly to Flora but places living actors instead of static props.

Each layer has:
- **Archetype** — the creature/NPC type to spawn
- **Distance bands** from buildings — near/far placement relative to structures
- **Time-of-day gate** (optional) — restrict spawns to baked day or night conditions
- **Max Fauna Actors** (global budget) — hard cap on total spawned fauna across all layers

<Note>
Movement settings in the Fauna layer are **Blueprint hints only** — OmniScape writes them as parameters on the spawned actor, but the actor's Blueprint is responsible for reading and acting on them. OmniScape does not implement movement or AI directly.
</Note>

## Ambient Audio (Beta)

OmniScape places **audio zones** across the settlement based on zone kind. Each kind maps to a real-world audio context.

| Zone kind | Typical use |
|-----------|-------------|
| **Interior** | Enclosed buildings, great halls, cellars |
| **Edge** | Settlement boundary, open countryside bleed-in |
| **Road** | Street ambience, crowd noise sampled along splines |
| **Node** | Market squares, plazas, focal points |

**Radius** on each audio zone drives real Unreal attenuation falloff — it is not decorative. Road-type layers sample positions along road splines rather than placing discrete spheres.

### Typical radius bands

| Read | Radius (cm) | Road spacing (cm) |
|------|-------------|-------------------|
| Intimate interior | 1,500 — 5,000 | — |
| Town ambient | 5,000 — 12,000 | 800 — 2,000 |
| Open edge / nature | 8,000 — 20,000+ | — |

An optional **Attenuation Override** per zone lets you swap in a custom attenuation asset when the default falloff doesn't match your audio setup.

<Warning>
**Ambisonic audio modes require a MetaSound source asset.** If you assign an ambisonic sound without a MetaSound source, OmniScape falls back to positional audio. Set up your MetaSound source first if you're targeting ambisonic spatialisation.
</Warning>

## PCG Hand-off (Beta)

OmniScape can export its layout data to Unreal's **Procedural Content Generation** framework for use in external PCG graphs.

The exported point sets include:
- Settlement nodes
- Road spline points
- Wall spline points
- Building footprints
- Settlement bounds

**To enable:**

<Steps>
  <Step title="Enable Beta mode">
    Click the Beta toggle in the status bar (bottom-right of the editor).
  </Step>
  <Step title="Toggle the data provider">
    Enable **Enable PCG Data Provider** on the generator.
  </Step>
  <Step title="Generate">
    Click Generate (or Refresh). OmniScape writes the layout data to the **UOmniScapePCGDataComponent** on the generator actor.
  </Step>
  <Step title="Connect your PCG graph">
    Reference the component from your PCG graph. Starter JSON contracts describing the point set schemas ship in **Content/PCG/**.
  </Step>
</Steps>

<Note>
The PCG Hand-off refreshes every time you generate. If your PCG graph caches data between generations, trigger a manual refresh on the PCG component after generating.
</Note>
