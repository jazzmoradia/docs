This page is the authoritative numeric reference for every control exposed in OmniScape's Control Center and generator Details panel. Each section lists the valid range, default value, and a brief note on what the control does. Use it alongside the Creative Directions guide — which explains *why* you'd want a given value — and the Control Center guide — which explains *where* each control lives in the UI. This page focuses purely on *what* the editor allows.

<Note>
  All linear distances are in **Unreal Engine centimetres (cm)**. At UE's default scale, 100 cm = 1 metre. Maximum building slope is in **degrees**. Angles, densities, weights, and normalized values (0–1 ranges) are dimensionless.
</Note>

<Note>
  The Creative Directions and Control Center guides explain how to use these parameters together to achieve a specific look or density. This page is a lookup reference — use it when you need to know the exact boundary of a slider or the default you're diverging from.
</Note>

<Accordion title="Settlement globals">

These controls apply to the entire settlement and affect every sub-system.

| Control | Range | Default | Notes |
|---|---|---|---|
| Settlement radius | ≥ 500, no hard max | 10,000 | Radius in cm of the circular footprint. Larger values require more nodes to maintain road density. |
| Num nodes | 1 — 50 | 5 | Number of district anchor nodes. More nodes = more road intersections and district variety. |
| Districts | 1 — 16 | 1 | Number of independently styled districts. Requires Voronoi pattern or district-capable layout to take full effect. |
| Overlap padding | ≥ 0 | 100 | Minimum clearance in cm between placed building plots to prevent overlap. Increase for sparser, more breathable layouts. |
| Max building slope | 0° — 89° | 30° | Buildings are not placed on terrain slopes steeper than this angle. Lower values restrict placement to flatter ground. |
| Edge density (`SprawlEdgeDensity`) | 0 — 1 | 0 | Controls how densely buildings populate the outer edge of the settlement. `0` = no sprawl; `1` = maximum edge density. |

</Accordion>

<Accordion title="Layout patterns">

Each layout pattern has its own set of controls. Only controls belonging to the currently selected pattern have any effect — the others are stored but ignored during generation.

### Radial

Concentric rings of roads radiating from settlement nodes.

| Control | Range | Default | Notes |
|---|---|---|---|
| Ring count | 1 — 20 | 3 | Number of concentric road rings per node. |
| Ring spacing | ≥ 200 cm | 1,500 cm | Distance between consecutive rings. Smaller values produce denser, more urban layouts. |
| Regularity | 0 — 1 | 0.85 | `1` = perfectly circular rings; `0` = heavily distorted, organic-feeling rings. |

### Voronoi

Cell-based district partitioning that produces irregular but structured blocks.

| Control | Range | Default | Notes |
|---|---|---|---|
| Regularity | 0 — 1 | 0.15 | Higher values push Voronoi sites toward a regular grid; lower values produce more organic cell shapes. |
| District gap | ≥ 0 cm | 200 cm | Road-width clearance maintained between district boundaries. |
| District blocks | on / off | on | When on, Voronoi cells are subdivided into smaller building blocks. Turn off for open-plan layouts with fewer internal roads. |
| Road redundancy | 0 — 1 | 0.2 | Adds extra road loops within districts. See [Road redundancy](#road-redundancy) below. |

### Perlin

Noise-driven organic road networks with no explicit cell structure.

| Control | Range | Default | Notes |
|---|---|---|---|
| Feature size | 0 — 1 | 0.45 | Controls the frequency of noise features. Low values = large, sweeping road curves; high values = tight, chaotic streets. |
| Roughness | 0 — 1 | 0.55 | Controls noise amplitude. Higher values produce more dramatic deviations from straight paths. |
| Road redundancy | 0 — 1 | 0.2 | Adds extra road loops. See [Road redundancy](#road-redundancy) below. |

### Axial / Grid

Straight-line axis roads with optional parallel grid subdivision.

| Control | Range | Default | Notes |
|---|---|---|---|
| Axis count | 2 — 8 | 4 | Number of primary road axes radiating from each node. |
| Regularity | 0 — 1 | 0.8 | `1` = evenly spaced axes; `0` = irregular angular distribution. |
| Parallel grid | on / off | off | When on, adds a secondary grid of roads parallel to the primary axes, producing block-based city layouts. |

### Organic

Freeform roads built from spacing and clustering logic rather than geometric primitives.

| Control | Range | Default | Notes |
|---|---|---|---|
| Spacing | ≥ 100 cm | 1,500 cm | Target distance between road segments. Smaller values produce denser, more tangled networks. |
| Grouping | 0 — 1 | 0.6 | How strongly roads tend to bundle together. `1` = roads cluster tightly; `0` = roads spread evenly. |
| Road redundancy | 0 — 1 | 0.2 | Adds extra road loops. See [Road redundancy](#road-redundancy) below. |

### Road redundancy

Available on **Voronoi, Perlin, and Organic** patterns only.

| Control | Range | Default | Notes |
|---|---|---|---|
| Road redundancy | 0 — 1 | 0.2 | Probability that dead-end roads are connected back into the network, forming loops. `0` = pure tree topology; `1` = maximum loop count. Higher values increase realism but also poly count. |

</Accordion>

<Accordion title="Roads">

Controls road visual geometry and intersection appearance.

| Control | Range | Default | Notes |
|---|---|---|---|
| Road width | ≥ 50 cm | 400 cm | Functional road width used for plot placement calculations and collision. |
| Road mesh tiling | 0.01 — 20 | 1 | UV tiling multiplier along the road mesh length. Increase for smaller texture tiles; decrease for larger. |
| Road mesh width scale (visual) | 0.1 — 10 | 1 | Scales the rendered road mesh width independently of the functional road width. Use to add kerbs or gutters without affecting building placement. |
| Junction / intersection radius scale | 0.25 — 3 | 1 | Scales the generated intersection cap mesh radius. Increase for smoother wide-road junctions; decrease to tighten narrow street corners. |

### Road width reference bands

Use these bands as a starting point when choosing a road width to match a visual intent.

| Visual intent | Road width range (cm) |
|---|---|
| Footpath / trail | 80 — 250 |
| Town street | 300 — 700 |
| Main road / boulevard | 700 — 1,400 |

</Accordion>

<Accordion title="Buildings">

Controls building slot counts, plot geometry, scale, and spawn weighting.

| Control | Range | Default | Notes |
|---|---|---|---|
| Medium buildings per node | 0 — 100 | 8 | Target number of Medium slot buildings placed around each road node. |
| Small buildings per node | 0 — 200 | 16 | Target number of Small slot buildings placed around each road node. |
| Plot min frontage (roadside) | ≥ 200 cm | 400 cm | Minimum plot width along the road edge. Controls how tightly buildings pack along a street. |
| Plot max frontage (roadside) | ≥ 200 cm | 2,000 cm | Maximum plot width along the road edge. Large values allow wider gaps between buildings. |
| Plot depth | ≥ 100 cm | 1,200 cm | Distance of each plot measured away from the road edge. |
| Plot depth variation | 0 — 0.5 | 0.15 | Random fractional variation applied to plot depth per plot. `0` = all plots identical depth; `0.5` = up to ±50% variation. |
| Large / Medium / Small scale | 0.1 — 10 each | 1 | Uniform scale multiplier applied to all spawned actors in each slot. Does not affect plot sizes. |
| Spur length | ≥ 100 cm | 800 cm | Length of dead-end spur roads generated off main roads to access buildings set back from the street network. |
| Weight (any slot) | ≥ 0 | 1 | Relative spawn probability for an individual asset within its slot. Higher weight = more frequent selection relative to other assets in the same slot. |

### Building density reference

Use these bands as a starting point for per-node counts to match a desired density feel.

| Density feel | Medium / node | Small / node |
|---|---|---|
| Sparse | 2 — 6 | 4 — 12 |
| Typical | 6 — 12 | 12 — 24 |
| Busy | 12 — 30 | 24 — 80 |

</Accordion>

<Accordion title="Flora">

Controls vegetation, rock, and debris layers. Each layer has independent settings.

| Control | Range | Default | Notes |
|---|---|---|---|
| Flora samples per layer | 10 — 5,000 | 400 | Number of placement attempts per layer before distance and exclusion filtering. Higher values produce denser layers but increase generation time. |
| Layer density multiplier | 0.01 — 10 | 1 | Post-filter density scale. Values above `1` increase effective density; values below `1` thin the layer. |
| Min distance from buildings | ≥ 0 cm | per layer | Flora samples closer than this distance to any building are discarded. Set higher to keep greenery clear of structures. |
| Max distance from buildings | ≥ 0 cm | per layer | Flora samples farther than this distance from any building are discarded. Use to confine vegetation to building surroundings rather than open areas. |
| Layer scale multiplier | 0.1 — 10 | 1 | Uniform scale applied to all meshes in the layer at spawn time. |
| Kind | Vegetation / Rock / Debris | Vegetation | Categorises the layer for sorting, audio, and gameplay filtering. Does not affect visual output directly. |

<Tip>
  During layout iteration, keep **Flora samples per layer** low (50–100) to speed up regeneration. Raise it to final values (400–2,000+) only when reviewing a near-final layout, or when baking to avoid runtime overhead.
</Tip>

</Accordion>

<Accordion title="Walls">

Controls the perimeter wall system. Walls are built from the current road layout — generate the layout first before fine-tuning wall parameters.

| Control | Range | Default | Notes |
|---|---|---|---|
| Wall perimeter fit | 0 — 1 | 1 | How tightly the wall follows the settlement perimeter. `1` = hugs the outermost buildings; `0` = uses a convex hull approximation. |
| Wall control point count | 8 — 96 | 36 | Number of spline control points in the wall loop. Higher values allow more detail and tighter curves; lower values produce smoother, more stylized outlines. |
| Wall perimeter clearance | ≥ 0 cm | 0 cm | Outward offset applied to the wall perimeter. Use to push walls further from the outermost buildings. |
| Wall coverage (partial mode) | 0.05 — 1 | 0.75 | Fraction of the perimeter covered when using partial wall mode. `1` = fully enclosed; `0.05` = minimal scattered segments. |
| Wall height | ≥ 50 cm | 400 cm | Height of wall mesh instances. |
| Wall thickness | ≥ 10 cm | 200 cm | Thickness of wall mesh instances. |
| Wall mesh tiling | 0.01 — 20 | 1 | UV tiling multiplier along the wall mesh length. |
| Pillar spacing (fraction of loop) | 0.001 — 0.5 | 0.01 | Fractional distance between pillar placements, expressed as a proportion of the total wall loop length. Smaller values place pillars more frequently. |
| Evenly spaced gate count | 0 — 32 | 0 | Number of gates distributed evenly around the wall perimeter. `0` disables automatic gate placement. |

</Accordion>

<Accordion title="Ambient audio (beta)">

Ambient audio emitter placement is in beta. Controls are subject to change in future releases.

| Control | Range | Default | Notes |
|---|---|---|---|
| Emitter radius | configurable per layer | — | Radius of each audio emitter's attenuation sphere. |
| Emitter spacing (road-following) | configurable per layer | — | Distance between emitters placed along road paths. |

### Typical radius bands

These are starting points for emitter radius values to match common spatial zones in a settlement.

| Spatial zone | Typical radius range (cm) |
|---|---|
| Intimate interior (courtyard, alley) | 1,500 — 5,000 |
| Town ambient (main street, square) | 5,000 — 12,000 |
| Open edge / nature transition | 8,000 — 20,000+ |

### Typical road spacing bands

| Zone type | Typical emitter spacing along roads (cm) |
|---|---|
| Town ambient | 800 — 2,000 |

</Accordion>
